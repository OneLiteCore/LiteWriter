Приложение предоставляет встроенный механизм резервного копирования, который периодически сжимает данные ваших глав в файл .zip.

# Внутренние файлы резервных копий будут потеряны, если приложение удалить

Встроенный механизм сохраняет файлы резервных копий .zip в приватной папке приложения. Эта папка, как и остальные данные приложения, будет удалена при удалении приложения.

Поэтому не полагайтесь только на этот механизм для защиты ваших данных. Он не предназначен быть вашим единственным способом резервного копирования.

# Как это работает

Когда вы выходите из приложения или наступает запланированное время дня, приложение проверяет, есть ли изменения, которые ещё не были сохранены в резервной копии. Если они есть, запускается процесс резервного копирования. По умолчанию между двумя процессами резервного копирования проходит не менее 8 часов.

После создания нового zip-файла приложение проверяет старые данные резервных копий и при необходимости удаляет их. По умолчанию приложение хранит 20 последних файлов резервных копий, поэтому вам не нужно беспокоиться о том, что резервные копии займут слишком много места.

Если вы настроили другие службы резервного копирования, например Google Drive или резервное копирование в хранилище устройства, новый zip-файл также будет загружен туда.
