앱은 장 데이터를 주기적으로 .zip 파일로 압축하는 내부 백업 기능을 제공합니다.

# 앱을 제거하면 내부 백업 파일도 사라집니다

기본 제공 메커니즘은 백업 .zip 파일을 앱 전용 비공개 폴더에 저장합니다. 앱을 제거하면 이 폴더도 다른 앱 데이터와 함께 삭제됩니다.

따라서 데이터를 안전하게 보관하는 방법으로 이것만 의존하지 마세요. 이 기능은 유일한 백업 수단으로 사용하도록 설계된 것이 아닙니다.

# 작동 방식

앱을 떠나거나 매일 예약된 시간이 되면, 앱은 아직 백업되지 않은 변경 사항이 있는지 확인합니다. 있다면 백업 프로세스가 실행됩니다. 기본적으로 두 번의 백업 프로세스 사이에는 최소 8시간의 간격이 있습니다.

새 zip 파일이 만들어지면 앱은 오래된 백업 데이터를 확인하고 필요하면 삭제합니다. 기본적으로 앱은 최신 백업 파일 20개를 유지하므로, 백업 파일이 저장 공간을 너무 많이 차지할까 걱정하지 않아도 됩니다.

Google Drive나 기기 저장소 백업 같은 다른 백업 서비스를 이미 설정했다면, 최신 zip 파일도 그곳으로 업로드됩니다.
