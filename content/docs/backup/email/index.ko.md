이메일 백업 서비스는 사용자의 이메일 계정을 이용해 백업 파일을 첨부파일로 사용자 자신의 메일함에 보냅니다. 즉, 백업 파일이 이미 사용자가 직접 관리하는 이메일 계정 안에 머물기 때문에 더 나은 개인정보 보호와 백업 데이터에 대한 완전한 통제권을 제공합니다.

저희는 사용자의 개인정보를 매우 중요하게 생각합니다. 앱은 사용자의 이메일 주소를 절대 업로드하지 않으며, 비밀번호를 오용하거나 수집하거나 훔치는 일도 절대 없습니다.

# 이메일 비밀번호 대신 앱 비밀번호가 필요한 이유

대부분의 이메일 서비스 제공업체는 일반 이메일 비밀번호로 서드파티 앱이 로그인하는 것을 허용하지 않습니다. 이것은 비밀번호 유출과 계정 악용의 위험을 줄이는 데 도움이 되는 일반적인 보안 정책입니다.

따라서 보통은 이메일 계정 설정에서 서드파티 앱 비밀번호 또는 클라이언트 전용 비밀번호를 만든 다음, 일반 이메일 비밀번호 대신 그 비밀번호를 앱에 입력해야 합니다.

# 주요 이메일 서비스 제공업체의 도움말 문서

- Gmail 도움말 문서: https://support.google.com/mail/answer/185833
- QQ Mail 도움말 문서: https://wx.mail.qq.com/list/readtemplate?name=app_intro.html#/agreement/authorizationCode

# 로그인 매개변수 이해하기

- **IMAP 서버 주소**: 이메일을 수신하고 읽는 데 사용되는 서버입니다.
- **IMAP 포트**: IMAP가 사용하는 네트워크 포트입니다. 기본 IMAP 포트는 일반적으로 **993**입니다.
- **SMTP 서버 주소**: 이메일을 보내는 데 사용되는 서버입니다.
- **SMTP 포트**: SMTP가 사용하는 네트워크 포트입니다. 기본 SMTP 포트는 일반적으로 **465** 또는 **587**입니다.
- **STARTTLS**: 연결이 시작된 뒤 일반 연결을 암호화된 TLS 연결로 업그레이드하는 보안 기능입니다. SMTP 포트가 **587**로 설정되어 있으면 앱은 기본적으로 **STARTTLS**를 활성화하고 이를 필수로 사용합니다.
- **이메일 주소**: 백업 이메일을 보내고 동시에 받는 이메일 계정입니다.
- **서드파티 앱 비밀번호**: 외부 앱용으로 이메일 제공업체가 생성하는 비밀번호입니다. 대부분의 경우 여기에는 일반 이메일 비밀번호를 사용할 수 없는데, 이는 보안상의 이유로 제공업체들이 서드파티 앱의 직접 비밀번호 로그인을 보통 차단하기 때문입니다.
