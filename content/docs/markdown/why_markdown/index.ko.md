Markdown은 단순하고 가벼운 글쓰기 형식입니다. 무거운 문서 편집기나 Word 스타일의 리치 텍스트 문서 형식에 의존하지 않고도, 일반 텍스트만으로 제목, 강조, 목록, 이미지처럼 자주 쓰이는 다양한 서식을 표현할 수 있습니다. 이 페이지 자체도 Markdown으로 작성되었습니다.

# Markdown 문법은 어떻게 작동하나요

Markdown은 렌더링되기 전에도 읽기 쉽습니다. 아래는 자주 쓰이는 문법 예시입니다.

- `# Heading`은 제목을 만듭니다. 더 낮은 수준의 제목에는 `#` 기호를 더 많이 사용할 수 있습니다.
- `---`은 가로 구분선을 만듭니다.
- `**bold**`는 텍스트를 **굵게** 표시합니다.
- `*italic*`는 텍스트를 *기울임꼴*로 표시합니다.
- `![Name](Image URL link)`는 이미지를 삽입하는 문법입니다.
- `- item`은 목록 항목을 만듭니다.
- `~~text~~`는 ~~예전 텍스트~~처럼 취소선을 추가합니다.

문법이 매우 작고 직관적이기 때문에, Markdown은 작성자가 서식 도구보다 내용 자체에 집중하도록 도와줍니다.

# 왜 텍스트 파일인가

일반 텍스트는 텍스트를 저장하는 가장 단순한 형식이며, 바로 그렇기 때문에 매우 널리 사용됩니다. 거의 모든 글쓰기 소프트웨어, 클라우드 노트 도구, 오피스 제품군, 심지어 명령줄 도구까지도 텍스트 파일을 읽고 쓰고 활용할 수 있습니다.

수십 년 전 DOS 시스템에서 작성된 텍스트 파일은 오늘날 Android에서도 여전히 정상적으로 읽고 편집할 수 있습니다. 같은 이유로, 오늘 Android에서 작성한 텍스트 파일도 앞으로 수십 년이 지난 미래의 컴퓨터에서 계속 읽고 쓸 수 있을 가능성이 매우 높습니다.

이 앱은 사용자의 글을 `.txt` 텍스트 파일로 저장하므로, 데이터를 더 오래 보존하고 더 쉽게 옮길 수 있습니다. 나중에 모든 노트를 다른 클라우드 노트, 지식 베이스, 또는 다른 글쓰기 도구로 옮기고 싶을 때도 마이그레이션이 훨씬 쉬워지고, 많은 경우 한 번에 끝낼 수 있습니다.
