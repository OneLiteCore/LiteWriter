Markdown — це простий і легкий формат для письма. Він дозволяє використовувати звичайний текст, щоб виражати багато поширених елементів форматування, таких як заголовки, виділення, списки та зображення, без залежності від важких редакторів документів або форматів rich text у стилі Word. Ця сторінка також написана в Markdown.

# Як працює синтаксис Markdown

Markdown легко читати ще до рендерингу. Нижче наведено кілька поширених прикладів синтаксису:

- `# Heading` створює заголовок. Для нижчих рівнів заголовків можна використовувати більше символів `#`.
- `---` створює горизонтальну роздільну лінію.
- `**bold**` робить текст **жирним**.
- `*italic*` робить текст *курсивом*.
- `![Name](Image URL link)` — це синтаксис для вставлення зображення.
- `- item` створює елемент списку.
- `~~text~~` додає закреслення, наприклад ~~старий текст~~.

Оскільки синтаксис дуже компактний і прямий, Markdown допомагає авторам зосереджуватися на змісті, а не на інструментах форматування.

# Чому текстові файли

Звичайний текст — це найпростіший формат для зберігання тексту, і саме тому він так широко використовується. Майже всі програми для письма, хмарні нотатки, офісні пакети та навіть інструменти командного рядка можуть читати, записувати й використовувати текстові файли.

Текстові файли, написані десятки років тому в системах DOS, і сьогодні можна нормально читати та редагувати на Android. Так само дуже ймовірно, що текстові файли, написані сьогодні на Android, залишатимуться придатними для читання й запису на комп’ютерах майбутнього ще через багато десятиліть.

Цей застосунок зберігає ваші тексти як файли `.txt`, що робить ваші дані більш довговічними та портативними. Якщо колись ви захочете перенести всі свої нотатки в інший хмарний блокнот, базу знань або інший інструмент для письма, міграція буде значно простішою й часто зможе виконуватися в один крок.
